Bagisto Web, Mobile, API 통합 QA 프로젝트

Bagisto demo store를 기준으로 웹 storefront, Flutter 모바일 앱, GraphQL API를 하나의 QA 범위로 묶어 자동화했습니다. 단순 채널별 스모크 테스트를 넘어서, 같은 백엔드에 연결된 주문 데이터가 웹, 앱, API에서 일관되게 보이는지까지 검증했습니다.

핵심 포인트

앱 UI에서 실제로 주문을 생성하고, 완료 화면의 주문번호가 API 주문 목록과 일치하는 흐름까지 구현해 cross-channel 일관성을 확인했습니다.

21 총 통과 테스트 수
4 웹 storefront 스모크 테스트
8 GraphQL API 테스트 및 주문 생성 검증
9 모바일 및 계약 테스트로 cross-channel 일관성 검증

프로젝트 범위

Web (4 tests): Playwright 기반 스모크 테스트 - 홈, 상품 상세, 장바구니, 체크아웃 진입 검증
API (8 tests): GraphQL 기반 E2E 시나리오 - 회원가입, 로그인, 상품 조회, 장바구니 추가, 체크아웃, 주문 생성 및 조회
Mobile (4 tests): Appium 기반 Android 앱 테스트 - 로그인 UI, 홈/상품 렌더링, 장바구니 추가 자동화
Contract (5 tests): 크로스채널 일관성 검증 - 웹/앱/API 간 상품 데이터 및 주문 데이터 동기화 확인

기술 스택

Python 3.13 + pytest - 테스트 프레임워크 및 fixture 기반 테스트 데이터 관리
Playwright - 웹 storefront 자동화 (headless Chrome)
Appium + ADB + UIAutomator - Android 모바일 앱 자동화 및 안정화
GraphQL Client (requests) - 모바일 앱 백엔드 API 계약 검증
POM (Page Object Model) - 웹/모바일 페이지 객체 분리 및 재사용성 확보

검증 결과와 시각 자료

Playwright 기반 웹 테스트는 실제 스크린샷으로, API와 모바일 테스트는 플로우 다이어그램으로 정리했습니다. 총 21개 테스트가 모두 통과했으며, 크로스채널 일관성까지 검증했습니다.
