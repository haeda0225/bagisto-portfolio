수동 테스트 — 블랙박스 기반 테스트 케이스 설계

동등분할(EP)·경계값 분석(BVA) 기반 테스트 케이스 설계

내부 구현을 보지 않고도 기능 명세와 사용자 관점만으로 검증 조건을 도출했습니다. 동등분할로 유효·무효 입력을 나누어 테스트 수를 효율화했고, 경계값 분석으로 오류가 발생하기 쉬운 구간을 집중적으로 확인했습니다.

동등분할 (EP, Equivalence Partitioning)

같은 동작을 유발하는 입력을 하나의 그룹으로 묶고 대표값을 선택해 검증했습니다. 테스트 수는 줄이되, 기능 커버리지는 유지하는 데 초점을 맞췄습니다.

경계값 분석 (BVA, Boundary Value Analysis)

오류가 자주 발생하는 경계 구간(min, min+1, max-1, max, max+1)을 중심으로 테스트를 설계했습니다. 동등분할로 놓치기 쉬운 경계 조건을 보완하는 데 활용했습니다.

탐색적 테스트 — 문제 발견에서 전략 개선까지

사전에 정해진 스크립트에만 의존하지 않고, 실제 사용자 관점에서 서비스를 탐색하며 예외 동작을 확인했습니다. 발견한 이슈는 원인을 분리해 해석하고, 이후 테스트 전략과 자동화 설계에 반영했습니다.

발견 항목 문제 원인 전략 반영
이메일 인증 정책 신규 계정으로 웹 로그인 시 "Verify your email account first." 메시지가 노출됨 API 오류가 아니라 데모 서버의 이메일 인증 정책 웹 로그인 분기를 제거하고, API 기반 인증 흐름 중심으로 자동화를 재설계
상품 타입 분기 virtual·grouped·bundle 상품은 "Add to Cart" 동작이 일관되지 않음 상품 타입별 필수 옵션과 선택 조건이 서로 다름 테스트 케이스와 자동화 범위를 simple product 중심으로 한정
서버 데이터 비고정 홈 화면 노출 상품이 바뀌면서 상품명 하드코딩 기반 테스트가 실패함 데모 서버 상품 데이터가 고정돼 있지 않고 수시로 변경됨 특정 상품명 대신 selector 기반의 첫 번째 상품을 동적으로 선택하도록 변경
수량 입력 UX 오류 음수(-1)를 입력해도 오류 메시지 없이 수량이 자동 조정됨 클라이언트 측 입력 유효성 검증이 충분하지 않음 BUG-001로 기록하고, 이후 회귀 테스트 항목에 포함
BUG REPORT — BUG-001
장바구니 수량에 음수를 입력해도 사용자 피드백이 제공되지 않음
Minor Priority: Medium
재현 절차 1. 상품을 장바구니에 담는다
2. 장바구니 화면에서 수량 필드를 선택한다
3. "-1"을 직접 입력한 뒤 포커스를 이동하거나 Enter를 누른다
기대 결과 "유효하지 않은 수량입니다" 메시지를 노출하고, 기존 수량을 유지해야 한다
